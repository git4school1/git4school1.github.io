Introduction

This dataset is interesting because happiness is a core factor of everyones life and analyzing how different variables can affect them in different parts of the world could yield meaningful results. Some of the questions we aim to answer are:

  1. What are the key factors that affect happiness scores in different regions and years?
  2. How do those factors correlate with each other and overall with the happiness score?
  3. Is there a way we can predict the happiness score using these factors with machine learning models?

When trying to answer these quetsions, we would like to learn how different societal factors and perceptions affect happiness scores in different regions, as well as the effects happiness has on other societal factors (such as gdp or life expectancy). By analyzing trends, we can determine how these findings change over the course of the years that are represented by our dataset.

Analyzing this data is important for the general population as well as government bodies. We are able to benchmark progress throughout the years and mark things such as societal progression or regression. Identifying these trends can provide numerical data from various events or laws passed. With this we are able to evaluate policy effectiveness, predict future outcomes on the general population's happiness and views, and potentially increase global happiness.

# check if values are outliers using the innerquartile range
def count_outliers(column):
      first_quartile = column.quantile(0.25)
      third_quartile = column.quantile(0.75)
      IQR = third_quartile - first_quartile
      lower_bound = first_quartile - 1.5 * IQR
      upper_bound = third_quartile + 1.5 * IQR
      outliers = (column < lower_bound) | (column > upper_bound)
      outliers_count = outliers.sum()
      return outliers_count

outlier_df.apply(count_outliers)

ANOVA Test

Hypothesis test on whether the region of an person has a correlation with their happiness score. Assume alpha is .05

$H_{0}$: There is no difference between any of the region means

$H_{A}$: There is a difference between at least one of the region means

Primary Analysis

Overview:

We have decided to use a linear model utilizing Elastic Net regularization to predict the happiness_score for a given country. We decided on this model because it will allow us to understand which features have a greater impact on a country's happiness score (by analyzing the coefficients of each feature in the model). Additionally, we decided to utilize Elastic Net regularization to increase accuracy and make our model more robust to outliers (since our dataset has a lot of outliers).

We also decided to remove the column 'region' because it was simply too much data to handle. Even if we batched the data to train it or made a mini data set, the model would get stuck in training. We did still want to invlove the location of the happiness score, so we kept the country. This was a lot of data itself as we had to one hot encode it, but it was managable.

# See how different different degrees change the performance of the model

for degree in range(1, 3):
  # Create model and make initial predictions
  # Introduced tolerance because of convergance
  model = make_pipeline(PolynomialFeatures(degree=degree), ElasticNet(alpha=0.01, max_iter=10000, tol = 1e-3))
  model.fit(X_train, Y_train)
  Y_train_pred = model.predict(X_train)
  Y_test_pred = model.predict(X_test)
  # Calculate MSE and R^2 for the dataset
  mse_train = mean_squared_error(Y_train, Y_train_pred)
  mse_test = mean_squared_error(Y_test, Y_test_pred)
  r2_train = r2_score(Y_train, Y_train_pred)
  r2_test = r2_score(Y_test, Y_test_pred)
  results = ({
        "MSE Train": mse_train,
        "MSE Test": mse_test,
        "R2 Train": r2_train,
        "R2 Test": r2_test
    })
  # Print results
  print(f"DEGREE {degree}:")
  print(f"  MSE Train: {results['MSE Train']:.4f}")
  print(f"  MSE Test: {results['MSE Test']:.4f}")
  print(f"  R2 Train: {results['R2 Train']:.4f}")
  print(f"  R2 Test: {results['R2 Test']:.4f}")
  print()

For both degrees 1 and 2, our outputs are pretty good for both MSE and R2 score. In degree 2, MSE lowers and our R2 score increases, signifying that it is a better model. This is true for both the training and testing data.

Above degree 2, MSE increases and R2 decreases for the test data, and the opposite happens for the training data. This is a sign of overfitting. Google Colab also cannot run above degree 2 without using an external source of RAM, and it take exponentially longer to run it on another IDE.

Because of this, we chose to use degree 2 for our final model.

Assessment of Model Performance

As a reference:

  • r2 = 0: the model doesn't explain any of the variance in the dependent variable (happiness_score)

  • r2 = 1: the model explains all of the variance in the dependent variable (happiness_score)

  • 'low' MSE (close to 0): indicates that the model's predictions are close to the actual values

  • 'high' MSE: indicates that the model's predictions are far from the actual values

  • Note: what 'low' and 'high' mean in MSE is completely based on the context of the data

More on MSE and R2 scores: https://www.bmc.com/blogs/mean-squared-error-r2-and-variance-in-regression-analysis/
